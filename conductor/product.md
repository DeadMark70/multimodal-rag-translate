# Initial Concept
一個具備多模態理解與自我修正能力的學術研究助理系統，專為解決傳統 RAG 的視覺盲區與學術文獻處理痛點而設計。

# Product Guide - Multimodal Agentic RAG System

## 1. 產品願景 (Vision)
本專案是一個具備多模態理解與自我修正能力的「學術研究助理系統」。它不僅僅是一個簡單的問答機器人，而是一個能模擬人類研究員思維（規劃、執行、評估、修正）的智慧代理系統。
**系統專注於解決傳統 LLM 在處理大量文獻時的 Token 限制問題，透過精準的 RAG 檢索機制，從龐大資料庫中提取關鍵資訊，而非依賴暴力填充上下文。**

## 2. 目標用戶 (Target Users)
*   **大學生與研究生**：用於大學專題、論文研究，輔助文獻回顧與數據理解。
*   **學術研究人員**：協助處理大量學術論文，提取關鍵資訊並進行深度對比。
*   **學術系統驗證者**：本系統特別設計用於驗證「Agentic RAG + GraphRAG」與「單純 LLM」在學術場景下的效能差異。

## 3. 核心功能 (Core Features)
*   **Agentic Workflow**：包含 Planner, Executor, Evaluator, Synthesizer 等角色，能自主進行深度研究循環，並持久化完整的執行歷程與詳盡報告。
*   **多模態理解**：精確解析論文中的圖表、圖片與公式，打破傳統 RAG 的視覺盲區。
*   **證據衝突處理**：識別文獻間的觀點衝突，避免平均化答案，保留學術論證的嚴謹性。
*   **GraphRAG 功能**：建立學術知識圖譜，從全域視視角關聯文獻，解決資訊碎片化問題。
*   **研究歷史管理**：支援檢索與重新查看過往的深度研究會話，包含完整的計畫、執行步驟、引用來源與最終報告。
*   **PDF 翻譯與重製**：具備 OCR 能力，可將論文翻譯後重新生成為 PDF，並盡可能保留原始排版格式。
*   **繁體中文支援**：系統全面支援以繁體中文進行提問與產出研究報告。
*   **自動化評估管線 (Evaluation Pipeline)**：整合 Ragas 庫，支援跨模型與跨層級的消融實驗與性能基準測試。具備完整的執行軌跡記錄 (thought_process, tool_calls) 與上下文元數據分析，支援深度病理診斷。

## 4. 交付與互動方式 (Interaction & Delivery)
*   **後端 API 服務**：系統作為核心引擎，提供穩定、高效的 API 介面。
*   **網頁介面整合**：透過外部開發的 Web 介面，讓用戶能輕鬆上傳 PDF、觸發研究流程並查看結果。
