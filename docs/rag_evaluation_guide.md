# 📚 RAG 評估題庫管理與題目撰寫指南

本指南旨在標準化 RAG（檢索增強生成）系統的測試題目開發流程。高品質的測試題目（Golden Dataset）是準確衡量系統性能（如 RAGAS 指標）的核心。

## 📝 欄位詳細說明與格式建議

### 1. ID (可選)
*   **說明**：題目的唯一識別碼。
*   **建議**：若留空，系統會自動生成 UUID。若手動輸入，建議採用 `[分類簡稱]-[序號]` 格式（例如：`HR-001`）。

### 2. 問題 (Question) *
*   **核心目標**：模擬真實用戶的查詢，且不應包含「上下文洩漏」。
*   **最佳實踐**：
    *   **獨立性 (Standalone)**：問題應能獨立被理解，不需依賴先前的對話。
    *   **避免暗示**：不要使用「根據這份文件...」或「如文中所述...」等詞彙。
    *   **明確性**：避免過於模糊的問題（例如：不要問「它的規定是什麼？」，應問「2024 年員工休假補貼的規定是什麼？」）。

### 3. 長版標準答案 (Master Ground Truth) *
*   **核心目標**：作為「最終事實來源」，提供最完整、最具備細節的答案。
*   **最佳實踐**：
    *   包含背景、細節說明與例外狀況。
    *   此欄位主要用於人類審閱以及作為短版答案的備援。

### 4. 短版標準答案 (RAGAS-ready Ground Truth)
*   **核心目標**：**優化 RAGAS 評分精準度**。這在計算 `Answer Correctness` 和 `Context Recall` 時至關重要。
*   **最佳實踐**：
    *   **事實密集型**：精簡非必要的修飾詞，保留核心事實與數據。
    *   **結構化**：若答案包含多個要點，建議使用條列式。
    *   **一致性**：確保與源文件中的術語完全一致。

### 5. Key Points (關鍵要點 - 每行一點)
*   **核心目標**：將標準答案拆解為不可分割的原子事實（Atomic Facts）。
*   **格式建議**：
    *   每行一個事實。
    *   範例：
        - 補貼金額為 5000 元
        - 申請截止日期為 12 月 31 日
        - 僅限正職人員申請

### 6. RAGAS Focus (關注指標)
*   **核心目標**：告知評估器此題目主要測試系統的哪種能力。
*   **建議選項 (可多選)**：
    *   `faithfulness`：測試系統是否會「幻覺」（是否只根據檢索內容回答）。
    *   `answer_correctness`：測試答案與標準答案的事實重合度。
    *   `answer_relevancy`：測試答案是否直接切中要點。
    *   `context_recall`：測試檢索模組是否能找回所有必要的片段。
    *   `context_precision`：測試檢索結果的排序是否優良（正確片段是否排在前面）。

### 7. 分類 (Category) & 難度 (Difficulty)
*   **說明**：用於後續的「切片分析」，幫助識別系統在特定領域或高難度題目上的表現。
*   **分類範例**：綜合比較題, 視覺驗證題, 列表枚舉題, 細節查詢題, 跨文件多跳推理, 衝突證據裁決, 圖文交叉重建, 反例干擾細節題。
*   **難度範例**：Easy, Medium, Hard。

### 8. 來源文件 (Source Documents)
*   **格式**：檔名之間以「逗號」或「換行」分隔。
*   **建議**：必須是系統中已上傳的 PDF 檔名，這有助於追蹤 `Context Recall` 是否正確定位到該文件。

### 9. 測試目標 (Test Objective)
*   **說明**：描述此題目的測試動機（例如：測試對 2024 年新修訂條文的檢索能力）。

### 10. 需要跨文件推理 (Cross-document Reasoning)
*   **說明**：若答案需要同時從兩個或以上的 PDF 提取信息並進行整合，請選「是」。這會觸發更高級的評估分析。

---

## 🌟 RAGAS 最佳實踐 Check-list

1.  **問題獨立嗎？** 用戶在不知道有這些文件的情況下會這樣問嗎？
2.  **標準答案（短版）是否事實精確？** 是否包含所有必要的數據（如日期、金額、專有名詞）？
3.  **Key Points 是否夠原子化？** 每個 Point 是否只代表一個單一事實？
4.  **是否包含「負面測試」？** 偶爾加入一個文件中不存在答案的問題，測試系統是否會誠實回答「不知道」而非亂編。

## 💡 範例參考

| 欄位 | 填寫內容 |
| :--- | :--- |
| **問題** | 2024 年專案經理的遠端辦公補貼標準為何？ |
| **長版答案 (Master)** | 根據 2024 年最新福利規定，專案經理級別以上的員工每月可申請遠端辦公補貼，金額為新台幣 3000 元，主要用於支付寬頻上網費用，且不需檢附單據。 |
| **短版答案 (RAGAS-ready)** | 2024 年專案經理遠端辦公補貼為每月 3000 元，用於寬頻費用，不需檢附單據。 |
| **Key Points** | - 補貼對象為專案經理<br>- 金額為每月 3000 元<br>- 用途為寬頻費用<br>- 免檢附單據 |
| **RAGAS Focus** | faithfulness, answer_correctness |
| **分類 / 難度** | 行政福利 / Easy |
| **需要跨文件推理** | 否 |
